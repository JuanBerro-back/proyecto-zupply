Actúa como un Ingeniero de Software Principal y Arquitecto Frontend Experto. Estoy desarrollando un proyecto integral llamado "Zupply IA" en REPLIT usando React + Vite + Tailwind CSS + Lucide React.

Necesito que generes el código completo y funcional de la aplicación web. Esta plataforma es un sistema Multi-Rol (B2B) que conecta a Restaurantes con Proveedores, gestionando logística, inventarios, predicciones de IA y costeo.

### ARQUITECTURA DE ROLES Y MÓDULOS DEL PROYECTO:

#### 0. Módulo de Autenticación (Login):
- **Pantalla de Inicio de Sesión:** Una UI moderna y atractiva donde el usuario ingresa sus credenciales.
- **Enrutamiento Protegido:** La aplicación inicia aquí. Al iniciar sesión exitosamente (con datos simulados), el sistema define el rol (`restaurante` o `proveedor`) y redirige al panel correspondiente.
- **Cierre de Sesión:** Un botón global en la barra de navegación para hacer logout y volver a la pantalla de login, permitiendo probar ambos roles fácilmente.

#### A. VISTA DEL RESTAURANTE (Protegido por Rol):
1. **Dashboard y Contabilidad IA:** Métricas clave, contabilidad de egresos y predicciones de reabastecimiento.
2. **Recepción, Rastreo y Código de Barras:**
   - Mapa interactivo (`react-leaflet` con OpenStreetMap en **Bucaramanga**) para ver la ruta de los vehículos del proveedor.
   - Escáner de Código de Barras (input de texto) para recibir pedidos y sumar stock automáticamente.
3. **Inventario Inteligente con Fechas de Vencimiento:**
   - CRUD completo del stock. Cada producto muestra su fecha de vencimiento. 
   - Alertas de IA rojas para productos próximos a vencerse (sistema FEFO - First Expiring, First Out).
4. **Gestor de Menú y Costeo Gastronómico Exacto:**
   - CRUD de recetas. El usuario agrega ingredientes (con sus gramajes). El sistema calcula el `Costo Total de Producción`.
   - Al ingresar el Precio de Venta (PVP), se calcula el **Porcentaje de Costo de Alimentos (Food Cost %)**: `(Costo Total / PVP) * 100`. 
   - **Regla de Negocio Estricta:** Alerta verde si el Costo de Alimentos es ≤ 30% (Margen ideal del 70%+). Alerta roja si supera el 35%, sugiriendo subir el PVP para no perder dinero.

#### B. VISTA DEL PROVEEDOR (Protegido por Rol):
1. **Dashboard de Ventas:** Resumen de pedidos recibidos de los restaurantes, ingresos y métricas.
2. **Gestión de Pedidos Entrantes:** Panel tipo Kanban o tabla para Aceptar, Preparar y Despachar pedidos.
3. **Despacho, Flota y Rutas (Logística Activa):**
   - Vista para despachar pedidos. Selector para asignar un vehículo disponible (ej. "Moto 1 - Placa XYZ", "Furgón Refrigerado").
   - Mapa (`react-leaflet` en Bucaramanga) mostrando la ubicación de los restaurantes destino para planificar la ruta.
4. **Inventario de Despacho (con Vencimientos):**
   - CRUD para que el proveedor gestione su catálogo. Control obligatorio de fechas de vencimiento de lotes.
